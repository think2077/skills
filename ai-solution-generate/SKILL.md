# AI 解决方案文档生成

## 任务目标
- 本 Skill 用于：根据客户提供的项目背景、需求信息，生成结构完整、内容专业的 AI 解决方案文档
- 能力：
  1. 分析客户业务痛点和项目背景
  2. 梳理和分类客户需求
  3. 设计 AI 解决方案架构
  4. 制定技术选型和实施计划
  5. 评估项目预算和 ROI
- 触发：当用户需要生成 AI 项目解决方案、技术方案文档、项目提案时使用

## 前置准备
- 依赖说明：无特殊依赖，由智能体直接处理
- 非标准文件准备：无

## 操作步骤

### 步骤 1：信息收集与确认
1. 向用户确认以下关键信息：
   - 项目名称
   - 客户/企业名称
   - 撰写人姓名
   - 版本号（默认 V1.0）
   - 日期
2. 收集项目背景信息：
   - 客户概况（行业、规模、业务范围）
   - 业务痛点与挑战（需量化数据）
   - 项目发起背景（政策、竞争、战略等驱动因素）
   - 项目目标与期望（可量化指标）
3. 收集需求信息：
   - 调研方式与对象
   - 原始需求清单
   - 需求分类与优先级

### 步骤 2：方案设计
1. 根据需求设计整体解决方案架构
2. 划分功能模块（建议 2-4 个模块）
3. 为每个模块设计：
   - 技术架构
   - 业务流程（AS-IS 和 TO-BE 对比）
   - 用户故事（Before/After 对比）
   - 详细功能说明
   - 价值评估

### 步骤 3：技术选型
1. 确定技术栈（大模型、向量数据库、开发框架等）
2. 说明选型理由和备选方案
3. 设计数据安全与合规方案
4. 规划系统性能与可靠性

### 步骤 4：实施计划
1. 制定实施策略（敏捷迭代/分阶段交付）
2. 绘制项目路线图/甘特图
3. 明确项目团队与分工
4. 识别风险并制定应对策略
5. 定义交付与验收标准

### 步骤 5：预算评估
1. 估算一次性建设费用（按模块）
2. 估算年度持续运营费用
3. 计算 ROI 和投资回报

### 步骤 6：文档生成
1. 按照模板格式生成完整文档
2. 确保所有表格、列表格式正确
3. 输出到用户指定位置

## 灵活适配指南

### 文档结构裁剪原则

根据项目规模和用户需求的复杂度，动态调整文档结构：

| 项目类型 | 建议章节 | 可省略章节 | 说明 |
| --- | --- | --- | --- |
| **小型项目**（预算<50 万） | 一、二、三、五 | 四（技术选型可简化）、六（预算可简略） | 聚焦核心需求和解决方案 |
| **中型项目**（50-200 万） | 一、二、三、四、五、六 | 无 | 标准完整结构 |
| **大型项目**（>200 万） | 一、二、三、四、五、六 + 附录 | 无 | 可增加附录：术语表、参考资料等 |
| **技术咨询/方案** | 一、三、四 | 二（需求已明确）、五、六 | 聚焦技术选型和架构设计 |

### 模块数量适配

- **简单场景**（1-2 个核心需求）：可合并为单一"解决方案"章节，无需细分模块
- **标准场景**（3-5 个需求）：建议划分为 2-3 个功能模块
- **复杂场景**（5+ 需求）：按业务域划分 4+ 模块，可增加"模块依赖关系图"

### 用户故事简化规则

| 场景 | 用户故事处理方式 |
| --- | --- |
| 内部技术评审 | 可省略用户故事，直接说明功能和技术指标 |
| 高层汇报/商务提案 | 保留 Before/After 对比，突出业务价值 |
| 技术团队交付文档 | 简化为"使用场景"说明，1-2 句话带过 |

### 预算部分灵活处理

- **仅提供技术方案时**：可省略第六章，或在第三章末尾简要说明"建设成本主要由 XX 构成"
- **需要商务报价时**：保留完整预算章节，费用区间可适当放宽（如±30%）
- **框架协议/意向书**：仅列出费用项类别，不填具体数字

### 技术选型详略程度

- **客户已指定技术栈**：仅需说明"遵循甲方技术规范"，列举关键组件即可
- **需要技术论证**：展开 4.2 节，详细说明候选方案对比和选型理由
- **纯 SaaS 方案**：可合并 4.1 和 4.3，重点说明数据安全和 SLA

## 使用示例

### 示例 1：快速方案（半日交付）
```
用户输入：帮我把这个需求转成解决方案，客户是做电商的，想搞个智能客服
输出：聚焦"智能客服"单模块，省略技术选型细节，预算给区间
```

### 示例 2：完整提案（3-5 日）
```
用户输入：我们要投标 XX 集团的 AI 项目，需要完整的解决方案文档
输出：按照完整模板结构，包含所有六个章节，技术选型详细论证
```

### 示例 3：技术架构文档
```
用户输入：客户已经确定合作了，现在需要输出详细的技术实施方案
输出：重点展开第三章（解决方案架构）和第四章（技术选型），简化背景和预算
```
