# 飞书 AI 解决方案生成

## 任务目标

本 Skill 用于：作为飞书三方服务商/ISV，在研发介入前，根据客户需求快速生成一份**面向业务决策者**的初步解决方案文档。

核心定位：
- **受众**：客户业务决策层（非技术人员）
- **目的**：确认需求理解一致、展示专业能力、框定项目范围
- **阶段**：售前/初步接洽阶段，还没到研发介入

## 核心原则

1. **用客户的业务语言写**——方案是给业务决策者看的，不用技术术语
2. **少即是多**——只保留用户指定的五个部分，核心是快速确认方向
3. **埋钩子**——对有争议或模糊的需求，标注"待详细调研确认"
4. **飞书特色要体现**——Aily智能伙伴原生嵌入（不是独立AI平台），知识问答跨产品搜索、多维表格灵活性、文档协作体验、集成平台编排能力
5. **与钉钉方案的差异要突出**——如果客户在飞书和钉钉之间选择，方案需隐含展示飞书优势（协作+AI原生融合 vs 低代码+组织管理）

## 前置准备

- 依赖说明：无特殊依赖
- 输入信息：
  - 客户名称与行业
  - 客户当前飞书使用情况（用了哪些产品、深度如何）
  - 核心需求描述（可以是非结构化的对话/笔记）
  - 已知的客户背景（规模、现有系统、痛点等）
- 如果客户当前不在飞书上，需在方案中增加"为什么选飞书"的简短论证

## 操作步骤

### 步骤 1：读取参考资源

**必须先读取以下两个文件**：

1. `references/outline.md` — 5 部分精简大纲模板
2. `references/feishu-capabilities.md` — 飞书能力矩阵速查表（含与钉钉的对照）

### 步骤 2：需求理解与分析

1. 简要梳理客户当前业务流程，标出主要角色、关键节点和现存问题
2. 列出涉及的角色及其飞书使用场景和痛点
3. 对模糊需求标注"待详细调研确认"

### 步骤 3：方案设计

1. 在“解决方案概要”中配置一张方案架构图，展示业务场景、核心功能模块、飞书能力组合和外部系统之间的关系
2. 按实际需求划分 2-4 个核心功能模块，不为凑数量增加模块
3. 每个模块简要说明模块作用、解决的问题、主要业务流程和使用的飞书能力组合
4. 每个核心功能模块配置一张对应的模块流程图，图中只保留角色、关键步骤、飞书产品介入点和结果
5. 选择 1-3 个关键场景，按“角色 + 操作步骤 + 结果”进行简要走查

### 步骤 4：对接集成设计

1. 列出用到的飞书能力和外部系统
2. 简要说明对接目的、数据方向及待确认事项
3. 不展开技术接口、字段和实施细节

### 步骤 5：输出文档

1. 严格按 `references/outline.md` 的五部分结构输出，不增加其他章节
2. 文件名格式：`{客户简称}-飞书AI解决方案-V1.0.md`
3. 保持 Markdown 表格对齐

## 飞书 vs 钉钉方案的关键差异

出方案时需注意以下差异，避免写成"钉钉方案的飞书翻译版"：

| 维度 | 钉钉方案写法 | 飞书方案写法 |
| --- | --- | --- |
| AI能力 | "在DEAP平台搭建智能体" | "飞书Aily原生嵌入文档/消息/表格，同一AI在多场景复用，且知识问答能跨产品搜索" |
| 表单/数据 | "用宜搭搭建表单" | "用多维表格搭建数据库，支持表格/看板/甘特多视图切换" |
| 流程 | "宜搭+审批流" | "飞书应用引擎搭建应用 + 审批 + 集成平台编排自动流" |
| 会议/录音 | "钉钉A1录音卡"或"会议录制" | "飞书妙记自动录制+转写+AI提取待办" |
| 集成 | "连接器" | "飞书集成平台（iPaaS）拖拽编排，更灵活" |
| 核心叙事 | 钉钉让组织管理更高效 | 飞书让协作和知识流动更顺畅 |

## 注意事项

- **读取顺序**：生成方案前必须先读取 `references/outline.md` 和 `references/feishu-capabilities.md`
- **飞书产品名要准确**：是"多维表格"不是"智能表格"，是"应用引擎"不是"宜搭"，是"妙记"不是"录制"
- **AI叙事不同**：飞书AI是"Aily"，含智能伙伴(个人级)/妙搭OpenClaw(极客级)/自定义智能体(团队级)三层。不是独立平台，原生嵌入各产品。方案中根据场景选择合适的层级——个人提效用智能伙伴，团队场景用自定义智能体
- **不要生搬钉钉方案结构**：如客户沟通主要在飞书消息+群，不是企业微信；数据管理用多维表格不是宜搭表单
- **信息不完整时**：主动向用户确认（特别是客户当前飞书使用情况）
- **不要增加其他内容**：除方案架构图和模块流程图外，不输出目标流程、价值总览、报价、实施计划、成功案例、公司资质和下一步行动
- **配图要求**：方案架构图必须有且仅有一张；每个核心功能模块必须有一张对应流程图。图片生成不可用时采用 Mermaid
