# 钉钉 AI 解决方案生成

## 任务目标

本 Skill 用于：作为钉钉三方软件服务商，在研发介入前，根据客户需求快速生成一份**面向业务决策者**的初步解决方案文档。

核心定位：
- **受众**：客户业务决策层（非技术人员）
- **目的**：确认需求理解一致、展示专业能力、框定项目范围
- **阶段**：售前/初步接洽阶段，还没到研发介入

## 核心原则

1. **用客户的业务语言写**——方案是给业务决策者看的，不用技术术语
2. **少即是多**——控制在 PPT 15-20 页的量级，核心是确认方向
3. **埋钩子**——对有争议或模糊的需求，标注"待详细调研确认"，为后续商务谈判留空间
4. **钉钉特色要体现**——免安装、组织通讯录打通、移动端原生体验、宜搭低代码

## 前置准备

- 依赖说明：无特殊依赖，由智能体直接处理
- 输入信息：
  - 客户名称与行业
  - 核心需求描述（可以是非结构化的对话/笔记）
  - 已知的客户背景（规模、现有系统、痛点等）

## 操作步骤

### 步骤 1：读取参考资源

**必须先读取以下两个文件**，确保输出结构和钉钉能力引用准确：

1. `references/outline.md` — 8 章节标准大纲模板
2. `references/dingtalk-capabilities.md` — 钉钉能力矩阵速查表

### 步骤 2：需求理解与映射

1. 从用户提供的原始信息中提炼需求，做"客户原话 → 理解后需求 → 应对思路"的映射表
2. 明确项目范围边界：**做什么 / 不做什么（本期）**——《边界比功能列表更重要》
3. 对模糊需求标注"待详细调研确认"

### 步骤 3：方案设计

1. 画出整体架构蓝图（一张图说明系统在钉钉生态中的位置）
2. 按 2-4 个模块划分功能，每个模块：
   - 说清楚干什么
   - 说清楚为什么这么做
   - 说清楚怎么做（业务语言）
   - 标注关键设计决策（如"为什么不用AI自动建联"）
3. 选 2-3 个核心场景做走查（步骤+示例）
4. 方案价值总览（改造前 vs 改造后对比表）

### 步骤 4：对接集成设计

1. 列出用到的钉钉能力及对应场景（宜搭/消息/知识库/DEAP/通讯录）
2. 列出外部系统对接清单（企业微信/公域平台/客户现有系统）
3. 画出数据流向简图

### 步骤 5：实施与商务

1. 实施策略：分几期、每期干什么、为什么是这个顺序
2. 粗粒度关键里程碑——**具体时间周期留空**，仅描述阶段顺序和依赖关系
3. 双方配合事项
4. 报价区间——**具体金额留空**，仅说明计价维度和影响因素
5. 投入产出参考——**回本周期等具体数字留空**
6. 风险与应对

### 步骤 6：输出文档

1. 按 `references/outline.md` 的 8 章节结构输出（不含 8.1 成功案例、8.2 公司资质）
2. 输出到用户指定的工作目录，文件名格式：`{客户简称}-AI解决方案-V1.0.md`
3. 保持 Markdown 表格对齐

## 注意事项

- **读取顺序**：生成方案前必须先读取 `references/outline.md` 和 `references/dingtalk-capabilities.md`
- **信息不完整时**：主动向用户确认关键信息（项目范围、优先级、预算预期），不可臆测填充
- **不要写技术实现细节**：这是初步方案，不是技术设计文档
- **钉钉能力引用要具体**：不要说"使用钉钉能力"，要说"用宜搭搭建商机线索表单"、"通过钉钉消息通知到责任人"
- **报价给区间不给固定价**：预留议价和需求变更空间；**具体金额、时间周期数字一律留空**
- **不写成功案例和公司资质**：8.1 成功案例参考、8.2 公司资质均不输出
- **每个模块的"Why"比"What"重要**：解释为什么选这个方案、为什么不做全自动，比罗列功能更能建立信任
