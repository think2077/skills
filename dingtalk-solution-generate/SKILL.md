# 钉钉 AI 解决方案生成

## 任务目标

本 Skill 用于：作为钉钉三方软件服务商，在研发介入前，根据客户需求快速生成一份**面向业务决策者**的初步解决方案文档。

核心定位：
- **受众**：客户业务决策层（非技术人员）
- **目的**：确认需求理解一致、展示专业能力、框定项目范围
- **阶段**：售前/初步接洽阶段，还没到研发介入

## 核心原则

1. **用客户的业务语言写**——方案是给业务决策者看的，不用技术术语
2. **少即是多**——只保留用户指定的五个部分，核心是快速确认方向
3. **埋钩子**——对有争议或模糊的需求，标注"待详细调研确认"，为后续商务谈判留空间
4. **钉钉特色要体现**——免安装、组织通讯录打通、移动端原生体验、宜搭低代码

## 前置准备

- 依赖说明：无特殊依赖，由智能体直接处理
- 输入信息：
  - 客户名称与行业
  - 核心需求描述（可以是非结构化的对话/笔记）
  - 已知的客户背景（规模、现有系统、痛点等）

## 操作步骤

### 步骤 1：读取参考资源

**必须先读取以下两个文件**，确保输出结构和钉钉能力引用准确：

1. `references/outline.md` — 5 部分精简大纲模板
2. `references/dingtalk-capabilities.md` — 钉钉能力矩阵速查表

### 步骤 2：需求理解与分析

1. 简要梳理客户当前业务流程，标出主要角色、关键节点和现存问题
2. 列出涉及的角色及其核心场景和痛点
3. 对模糊需求标注"待详细调研确认"

### 步骤 3：方案设计

1. 在“解决方案概要”中配置一张方案架构图，展示业务场景、核心功能模块、钉钉能力和外部系统之间的关系
2. 按实际需求划分 2-4 个核心功能模块，不为凑数量增加模块
3. 每个模块简要说明模块作用、解决的问题、主要业务流程和使用的钉钉能力
4. 每个核心功能模块配置一张对应的模块流程图，图中只保留角色、关键步骤、钉钉能力介入点和结果
5. 选择 1-3 个关键场景，按“角色 + 操作步骤 + 结果”进行简要走查

### 步骤 4：对接集成设计

1. 列出用到的钉钉能力和外部系统
2. 简要说明对接目的、数据方向及待确认事项
3. 不展开技术接口、字段和实施细节

### 步骤 5：输出文档

1. 严格按 `references/outline.md` 的五部分结构输出，不增加其他章节
2. 输出到用户指定的工作目录，文件名格式：`{客户简称}-AI解决方案-V1.0.md`
3. 保持 Markdown 表格对齐

## 注意事项

- **读取顺序**：生成方案前必须先读取 `references/outline.md` 和 `references/dingtalk-capabilities.md`
- **信息不完整时**：主动向用户确认关键信息（项目范围、优先级、预算预期），不可臆测填充
- **不要写技术实现细节**：这是初步方案，不是技术设计文档
- **钉钉能力引用要具体**：不要说"使用钉钉能力"，要说"用宜搭搭建商机线索表单"、"通过钉钉消息通知到责任人"
- **不要增加其他内容**：除方案架构图和模块流程图外，不输出目标流程、价值总览、报价、实施计划、成功案例、公司资质和下一步行动
- **配图要求**：方案架构图必须有且仅有一张；每个核心功能模块必须有一张对应流程图。图片生成不可用时采用 Mermaid
- **每个模块的"Why"比"What"重要**：解释为什么选这个方案、为什么不做全自动，比罗列功能更能建立信任
