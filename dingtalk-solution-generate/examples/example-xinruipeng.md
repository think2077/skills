# 新瑞鹏宠物医疗集团 AI 智能化解决方案（初步方案）

---

**客户名称：** 新瑞鹏宠物医疗集团有限公司
**版本：** V1.0（初步方案）
**日期：** 2026年5月15日

---

## 一、项目概述

### 1.1 项目背景

新瑞鹏是国内宠物医疗连锁龙头，旗下数百家医院，在AI应用上已有探索——已上线AI语音录入、智能辅助诊断等功能，覆盖200+病种。但实际落地中遇到几个突出问题：

- 医生端的AI语音录入激活率不到60%，功能有了但没人用
- 小红书、微博、美团等公域平台上大量客户声音没人管，商机白白流失
- 内容营销靠人工写、人工发，产出频率跟不上
- 现有的客服AI能力弱，图片视频处理不了，回复也生硬
- 企业微信有很多客户群要维护，人力根本顾不过来

### 1.2 建设目标

用钉钉生态把"客户从哪里来→怎么接住→怎么服务好→怎么持续触达"这条线打通。本次重点解决四个问题：

1. 把AI语音录入用起来（医生不用手动开了）
2. 公域+私域的信息自动收集、分类、跟进
3. 营销内容能批量生成、一键发布
4. 客服AI升级，能区分营销/售前/售后三个场景，还要能看图看视频

### 1.3 项目范围

| 做什么 | 不做什么（本期） |
| --- | --- |
| 私域消息AI分类与自动回复 | AI自动建联（风险大，需人工介入） |
| 公域舆情监控与商机线索收集 | 全自动无人值守内容发布（所有内容需人工审核） |
| 营销文章AI生成 + 多平台分发 | 家庭智能设备接入（远期目标） |
| 三大场景智能客服（营销/售前/售后） | 诊室视频监控硬件改造（成本高，放二期评估） |

## 二、需求理解与分析

### 2.1 业务流程梳理

**当前状态（AS-IS）：**

```
公域客户（小红书/微博/美团）
    → 评论、私信没有人盯，偶尔运营刷一下
    → 看到像商机的，手动记在Excel或脑子里
    → 过几天忘了，没人跟进

私域客户（企业微信/自有账号）
    → 消息分散在各个平台，客服一个个人工回
    → 回复质量参差不齐，话术不统一
    → 图片视频消息看不懂，只能回"请到店咨询"

内容运营
    → 运营自己找选题（刷小红书看别人发什么）
    → 自己写文章、找图片 → 排版 → 逐个平台发布
    → 一周能出1-2篇就不错了
```

**目标状态（TO-BE）：**

```
公域客户
    → 舆情系统24小时自动扫描相关文章和评论
    → AI判断是商机/客诉/咨询，自动分到对应表格
    → 商机钉钉通知责任人 → 人工跟进建联

私域客户
    → 所有平台消息统一接入，AI秒级分类
    → 常规咨询AI自动回复；客诉和商机自动通知对应的人
    → 图片/视频AI初步分析后，再决定是否转人工

内容运营
    → AI自动抓取热点选题 → 运营选一下 → AI生成初稿
    → 人工审核修改 → 一键发布到企微/小红书/朋友圈
    → 每天都能发，内容质量还稳定
```

### 2.2 角色与场景

| 角色 | 核心场景 | 痛点 |
| --- | --- | --- |
| 运营人员 | 每天盯公域舆情、找商机线索、写文章发内容 | 平台多、信息散、效率低 |
| 客服人员 | 回企业微信私信、处理客诉、维护客户群 | 重复问题多、图片视频处理不了、群太多 |
| 医生 | 就诊时用AI语音录入辅助写病例 | 忙起来忘了开，功能入口不够明显 |
| 管理层 | 看整体运营效果、商机转化情况 | 数据散在各处，看不到全貌 |

### 2.3 关键需求映射

| 客户原话 | 理解后的需求 | 应对思路 |
| --- | --- | --- |
| "AI语音录入医生总是忘了开" | 不是功能问题，是触发时机问题 | 系统主动提醒+预约联动触发，不动现有功能 |
| "公域评论信息抓到线索" | 小红书、微博、美团的评论/文章需自动采集分析 | 舆情监控+AI分类→宜搭表单→钉钉通知 |
| "私域账号评论私信AI分析分类" | 企微等自有渠道消息统一管理，自动分类处理 | 消息接入→AI三类分级（客诉/商机/咨询）→分别处理 |
| "生成文章发布到小红书" | 从热点发现到内容生成到发布一体化 | 热点抓取→素材库→AI生成→人工审核→多平台发 |
| "营销客服、售前客服、售后客服" | 不同场景需要不同的AI，不能一个AI应付所有 | 知识库底座+三个场景智能体，各管各的 |
| "客户发图片视频AI处理不了" | 需要多模态识别能力 | 图片识别+视频理解，结果作参考不替代医生判断 |

> **待详细调研确认：** 各平台日均消息量、现有客服团队规模与排班、HIS/预约系统接口开放情况、视频监控硬件现状

## 三、解决方案概要

### 3.1 整体架构蓝图

```
┌────────────────────────────────────────────────┐
│                  业务场景                        │
│   诊室录音  │  舆情商机  │  内容营销  │  智能客服  │
├────────────────────────────────────────────────┤
│              钉钉DEAP平台                   │
│        知识库 │ 智能体编排 │ 权限管控             │
├────────────────────────────────────────────────┤
│              钉钉原生能力                        │
│   宜搭(表单) │ 消息通知 │ 知识库 │ 组织通讯录     │
├────────────────────────────────────────────────┤
│              外部接入                            │
│   企业微信 │ 小红书API │ 微博 │ 美团 │ HIS系统    │
└────────────────────────────────────────────────┘
```

一句话解释：**客户在哪，数据就接到哪；数据进来，AI自动分类处理；需要人介入的，钉钉通知到人；AI能自己处理的，自动回复不掉链子**。全部跑在钉钉上，不用另外装App。

### 3.2 核心功能模块

#### 模块一：诊室AI语音录入优化

医生不是不想用，是忙起来忘了开。核心思路是**把"主动开"变成"被动提醒"**：

- 预约系统联动：有预约时，就诊时间到了自动推消息给医生，附"一键开启录音"入口
- 系统通知：医生进入接诊界面时，钉钉弹窗提醒"是否开启AI语音录入？"
- 录音归档：就诊结束自动保存，医生在病例界面直接选择录音片段填入

> **方案对比：** 曾评估过诊室视频监控自动开录音的方案，但涉及摄像头改造，成本较高。本期优先走"系统通知+预约联动"的低成本路径，后续视效果再决定是否上视频方案。

#### 模块二：全域舆情与商机线索管理

**公域（小红书、微博、美团等）：**

```
配置关键词（品种名/病症/宠物医院推荐等）
  → 舆情系统24小时扫描公域文章和评论
  → AI分析是不是商机（比如"求推荐靠谱猫医院"就是）
  → 是商机 → 自动写入宜搭【商机线索表】→ 钉钉通知运营
  → 人工核实 → 主动建联
```

**私域（企业微信、各平台自有账号）：**

```
所有平台的评论/私信统一接入
  → AI分析分类：
    ├── 客诉（不满、投诉） → 宜搭客诉表 → 通知客服主管优先处理
    ├── 商机（询价、意向） → 宜搭商机表 → 通知销售跟进
    └── 咨询（产品、价格、服务问询） → AI自动回复
```

> **为什么不用AI自动建联？** AI在公域主动去私信别人，容易被平台判定为营销号封禁，品牌风险也大。现阶段"AI发现→人工建联"是最稳妥的做法。
>
> **为什么私域自动回复只做咨询类？** 客户投诉和明确意向需要人去判断和处理，AI直接回容易出问题。

#### 模块三：宣传文章生成与自动发布

核心逻辑：**AI学别人爆款的内容模式，结合自己的素材，生成自己的内容，人审核后发**。

| 环节 | 做什么 |
| --- | --- |
| 热点发现 | AI扫描小红书、抖音等宠物类热门内容，提取高互动选题（如"幼犬到家一周注意事项"） |
| 素材管理 | 建一个后台素材库，把医院场景照片、医生介绍、案例、科普知识分类存好，AI自动打标签 |
| 内容生成 | 选定热点+选定素材 → AI生成初稿，支持多种风格（专业科普/温馨故事/促销活动） |
| 人工审核 | 所有内容必须人工审过才能发——避免AI"胡说八道"影响品牌 |
| 一键发布 | 审核通过后，一键推到企业微信朋友圈、小红书、企微客户群 |

> **待详细调研确认：** 小红书开放API权限是否支持自动发布、朋友圈推送的企业微信接口限制

#### 模块四：客户AI智能体

这是整个方案的**底座**。上面的模块二、模块三都依赖这里的能力。

**第一步：建知识库**

把该整理的知识先整理进钉钉知识库——服务项目、价格、常见宠物病症、护理常识、FAQ标准话术、医生介绍、就诊流程等。知识不准，后面AI全跑偏。

**第二步：搭三个场景智能体**

| 智能体 | 用在哪儿 | 干什么 | 举个例子 |
| --- | --- | --- | --- |
| 营销客服 | 企微社群、朋友圈 | 发养宠知识、活动推荐、活跃群气氛 | 在群里每天发一条"夏天猫咪中暑怎么办"科普+引导到店体检 |
| 售前客服 | 企微私信、平台私信 | 介绍服务、报价、引导预约、识别图片视频 | 客户发一张猫咪皮肤照片 → AI初步判断可能是猫癣 → 推荐附近医院皮肤科预约 |
| 售后客服 | 企微私信、400 | 就诊后随访、用药提醒、复查提醒、客诉处理 | 术后第3天自动发消息"宝贝恢复怎么样？有问题随时找我" |

**第三步：多模态能力**

客户发的图片和视频，AI要能看懂：
- 图片：品种识别、皮肤病/外伤初步判断、化验单OCR读取
- 视频：宠物异常行为（跛行、抽搐等）初步识别

> 多模态只给参考，不做诊断——最终判断权在医生手里。

### 3.3 关键场景走查

**场景一：公域发现一条商机到跟进**

> 运营小王的钉钉弹出一条消息："发现商机线索——小红书用户@xxx 评论'杭州滨江哪家宠物医院做绝育好啊求推荐'，意向度：高" → 点开消息直接跳到宜搭商机线索表，已经自动填好了平台、原帖链接、用户昵称、意向关键词 → 小王点"接单"，用企微去主动联系该用户 → 跟进结果记录回宜搭。

**场景二：客户在企微私信咨询**

> 宠物主人李女士在企微给医院发消息："我家布偶猫最近一直打喷嚏，怎么办"，附带一张猫咪鼻子照片 → AI识别：品种=布偶猫，图片分析=鼻腔无明显分泌物，症状关键词=打喷嚏 → AI回复："李女士您好，猫咪打喷嚏可能的原因有感冒、过敏、鼻支等。建议带来做个体检，我们滨江分院有专门的猫科门诊，需要帮您预约吗？" → 同时生成一条咨询记录存到宜搭咨询表 → 如果客户回复"转人工"，立刻转给在线客服。

### 3.4 方案价值总览

| 维度 | 现在 | 用了之后 |
| --- | --- | --- |
| 线索获取 | 人工刷平台，不定期 | 24小时自动扫，日均50+有效线索 |
| 线索遗漏 | 大量评论私信没跟进 | 全量接入，分类处理不遗漏 |
| 客户响应 | 工作时间人工回，排队久 | 咨询秒回，7×24在线 |
| 内容产出 | 一周1-2篇，质量看个人 | 一天多篇，质量稳定，人工把控 |
| 客服处理 | 图片视频看不懂 | AI初步分析，辅助判断 |
| 语音录入 | 40%+的就诊忘了开 | 系统提醒，预计漏开率降到15%以下 |

## 四、对接与集成方案

### 4.1 钉钉能力利用

| 钉钉能力 | 用在哪个场景 | 为什么用 |
| --- | --- | --- |
| **宜搭** | 商机线索表、客诉记录表、咨询记录表 | 低代码搭建，不用写代码，表单+流程半小时搞定，后期运营自己就能改 |
| **钉钉消息** | 商机通知、客诉告警、就诊录音提醒 | 直接到人，不用另外配短信/邮件 |
| **钉钉知识库** | 产品知识、医疗知识、FAQ话术库 | 与钉钉组织权限打通，谁该看什么自动控 |

| **组织通讯录** | 通知分派到具体责任人 | 按部门/角色自动匹配，不用手动维护分发名单 |

| 小红书 | 开放平台API | 双向：评论采集+内容发布 | P1 |
| 集团HIS系统 | 接口对接 | 双向：预约信息+录音归档 | P1 |
| 美团/大众点评 | API/爬虫 | 单向：评论/评价采集 | P2 |

- **付款方式：** 按里程碑节点分期支付（具体比例待商议）

### 6.2 报价区间

> 以下为方案阶段初步估算，实际价格在需求确认后以正式报价单为准。

| 费用类别 | 预估区间（万元） | 说明 |
| --- | --- | --- |

- 智能体调优深度（场景越多、准确率要求越高，调优投入越大）
